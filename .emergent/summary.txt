<original_problem_statement>
The user wants a comprehensive order management system for a restaurant/catering business. The application should be designed for tablets but also work on mobile.

**PRODUCT REQUIREMENTS (based on initial request and subsequent clarifications):**
- **Core Entities:** Manage Orders, Dishes, Customers, and daily Menus.
- **Dish Management:**
    - Create, update, and deactivate dishes.
    - Assign dishes to categories (e.g., Primi, Secondi, Contorni).
    - Manage categories (CRUD).
- **Menu Management:**
    - Create a Menu of the Day by selecting dishes and setting available portions.
- **Order Management:**
    - Create new orders for a specific customer.
    - Add dishes from the Menu of the Day to an order.
    - The system must track remaining portions and prevent overselling.
    - Orders must be editable at any time.
    - Orders should have multiple statuses (e.g., In Attesa, In Preparazione, Pronto, Completato, Consegnato, Annullato).
- **Customer Management:**
    - Basic CRUD for customer information.
- **Reporting:**
    - View daily totals, sales by channel, and sales per dish.
- **UI/UX:**
    - The application must be in Italian.
    - The interface must be intuitive and well-designed for touch interaction on tablets and mobile devices.
- **Future Features:**
    - PDF export of reports or orders.

**Initial Decisions:**
- **Authentication:** Deferred for now.
- **Database:** The agent chose MongoDB, and the user agreed.
- **Payments:** Not required; the app only needs to calculate totals.
- **Tech Stack:** FastAPI backend, Expo (React Native) frontend.
